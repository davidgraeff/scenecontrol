Der Quellcode kann direkt von GitHub bezogen werden. Unter Software bauen wird das selber kompilieren näher erläutert. Fertige Binaries von der letzten lauffähigen Version (trunk) des Kernprozesses, der Hilfsprogramme und einiger Plugins werden für aktuelle Ubuntu Distributionen angeboten.

Software bauen

Es wird CMake (www.cmake.org) benötigt. Im CMake-Gui Programm kann grafisch der Quellcodeordner und ein davon unabhängiger BUILD-Ordner angegeben werden. CMake versucht alle benötigten Abhängigkeiten und den angegebenen C++ Compiler zu finden. Nach dem Generierungsschritt liegen (je nach Auswahl) Visual Studio Projektdateien, Makefiles, usw vor.

Unter Windows: Hier wird VisualStudio 10+ empfohlen. Nicht alle Teile dieses Projekts, insbesondere nicht alle Plugins lassen sich unter Windows bauen.

Unter Linux: Hier bietet sich das Erstellen lassen von Makefiles an. Anschließend in den BUILD-Ordner wechseln und den Befehl "make" ausführen. Um ein Deb Packet generieren zu lassen den Befehl "make package" nutzen.

Plugins

SceneControl ist bewusst mit einem sehr leichtgewichtigem, funktionsarmen Kernprozess entwickelt worden (siehe Beschreibung der Architektur). Funktionalität wird über Plugins realisiert. Einige werden mitgeliefert, andere sind über Drittrepositories abrufbar.

Mitgeliefert

Einige Plugins befinden sich im selben Repository wie die Kernanwendung. Zum einen sind das generische Plugins, welche ohne weitere Voraussetzungen auf allen Systemen laufen und Basisfunktionen anbieten (Time, WOL) oder als Schnittstelle für andere Plugins dienen (Leds, Schalter). Zum anderen enthält das Repository einige Plugins für die Steuerung von frei verfügbarer, bekannter Software (Xbmc, ...) oder Hardware Kommunikationsprotokolle (DMX/ArtNet).

Drittanbieter Plugins

Fordern Plugins spezielle Hardware oder Software, werden diese nicht im SceneControl Repository aufgenommen. Liegen Drittanbieter Plugins im Quellcode vor, können diese einfach in das plugins/3rd-party Verzeichnis vor dem Bauvorgang kopiert werden. Das Buildsystem CMake bietet diese Plugins dann ebenfalls zum Kompilieren und Installieren an. Drittanbieter Binary Plugins werden nach der Installionsanleitung des Herstellers installiert.
